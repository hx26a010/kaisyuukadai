# 盤上遊戯 仕様書

## 1. 概要
- **ジャンル**: 難易度選択つき2Dテーブル対局ゲーム
- **ターゲット操作環境**: マウス操作（PC想定）、またはタッチ操作（スマートフォン想定）のみ

## 2. 進行方式と勝敗条件
- **初期手番（先攻・後攻）**: ゲーム開始時に、プレイヤーと相手のどちらかの「王（王将）」がランダムに「玉（玉将）」に変更されます。「玉」を所持している側が先攻（先手）として対局が開始されます。
- **対局形式**: プレイヤー（自分）と敵AI（相手）が交互に一手ずつ指し進める対局型（1手ごとに1カウント進む）
- **手数の上限**: 最大 **500手** とします。プレイヤーが1手動かしたら「+1カウント」、敵AIが1手動かしたら「+1カウント」とカウントされ、500手に達した時点で対局終了（引き分け）となります。
- **勝利条件**: 相手の王将を、自分の駒が動かせる範囲内で取る
- **敗北条件**: 自分の王将が相手（敵AI）に取られる

## 3. 難易度仕様
- **難易度のレベル**: 以下の **4段階** を用意します。
  - **簡単**（AIはランダムや単純な手を好む、思考時間は極めて短い）
  - **普通**（一般的な思考アルゴリズム、標準的な思考スピード）
  - **難しい**（先読み深さが増し最善手を狙う、思考時間はやや長い）
  - **プロ**（最も深い探索を行い最高強度の指し手を行う、思考時間は最適化の範囲内で長め）
- **難易度の影響範囲**: 難易度選択によって敵AIの「思考の強さ」および「思考の速さ」のみが変化します。
- **駒の制限**: 使用できる駒の種類や数は、どの難易度を選択しても変化しません（共通配置）。

## 4. 画面レイアウト・ステージ構成
- **ステージサイズ**: 9 × 9 の 81マス
- **マス目の等価性**: 1マス1マスの大きさを均等に分ける
- **画面表示**: 1画面に収まるサイズで表示する

## 5. 駒の仕様と配置
- **駒の種類**: 将棋の駒を採用する
- **移動規則**: 将棋の駒と同じ動き方ができる
- **配置**: 対戦相手の駒が自分の駒と対照的に配置される（将棋の初期配置と同等）
- **駒の「成る」ルール**: 
  - 敵陣（手前から数えて奥の3段）に駒が進んだとき、または敵陣から移動(または退出)したときに、駒を「成る（パワーアップ）」させることができます（選択ダイアログ、または自動処理）。
  - 成った駒（と金、竜王、竜馬、成銀、成香、成桂）は、将棋と同じ移動規則に変化します。

## 6. コアメカニクス（ゲームプレイ）
- **持ち駒の再配置（打つ）**: 自分で取った相手の駒（持ち駒）を、自分の番でステージ（盤面）の空いているマス内に再配置できる
- **特殊ルール（反則行為の禁止）**:
  - **二歩**: 自分の「と金」に成っていない「歩」が既に存在する縦列（筋）のマスに、新たに持ち駒の「歩」を配置することはできません。
  - **打ち歩詰め**: 持ち駒の「歩」を直接相手の「王将」の目の前に打って、即座に詰みの状態にすることはできません。
  - **行き所のない駒**: 駒を移動、または持ち駒から打つ際、その先で一切動かすことができなくなるような配置（例: 一番奥の行に「歩」や「香車」を配置する等）は反則とし、配置できません。
- **ゲームコントロール**: わざと自分の駒を相手に取らせるなどして戦局を有利に導き、相手の王将を取れるように思考します。
